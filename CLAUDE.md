# CLAUDE.md

このファイルはClaude Codeがこのリポジトリで作業する際の指針です。実装前に必ず `/docs/spec.md` を読み、機能要件・データモデル・フェーズ分割に従ってください。

---

## プロジェクト概要

Symbol(XYM)投げ銭機能付きブログサイト。ノンカストディアル・ウォレット（秘密鍵はブラウザのlocalStorageにパスフレーズ暗号化して保存、サーバーには一切送信しない）が最大の特徴。

---

## 技術スタック（厳守）

- Next.js (App Router) + TypeScript
- Tailwind CSS
- Tiptap（リッチテキストエディタ）
- NextAuth.js（メール認証）
- PostgreSQL + Prisma
- symbol-sdk / symbol-hd-wallets
- Web Crypto API（AES-256-GCM、ローカル鍵暗号化のみ）
- 画像: S3互換ストレージ（環境変数でエンドポイント切替可能に）

別のライブラリ・フレームワークへの変更は提案のみとし、無断で変更しない。

---

## 開発フェーズと進め方

`/docs/spec.md` の「7. 開発フェーズ分割」に従い、**1フェーズずつ**実装する。

1. 現在のフェーズをユーザーに確認、または `/docs/progress.md` を見て次のフェーズを判断する
2. そのフェーズのタスクのみを実装する（先のフェーズの機能を先取りして実装しない）
3. 実装完了後、`/docs/progress.md` に完了内容を追記する
4. ビルド・型チェック・lintが通ることを確認する
5. 次のフェーズに進む前に一度ユーザーに確認を取る

## 絶対に守るルール（セキュリティ）

### 秘密鍵・ニーモニック関連
- **秘密鍵・ニーモニックフレーズ・ウォレットパスフレーズをサーバーに送信するコードを書かない**（API routeに渡さない、ログに出力しない、DBに保存しない）
- これらはすべてクライアントサイド（ブラウザ）のメモリ or 暗号化済みlocalStorageにのみ存在する
- サーバーに保存してよいのは「公開アドレス」のみ
- Phase 5（ウォレット生成）・Phase 6（送金）の実装後は、**コミット前にdiffを提示し、人間のレビューを促す**（自動でcommit/pushしない）

### 暗号化
- ローカル保存の暗号化はWeb Crypto API (`crypto.subtle`) のみを使用する
- 独自の暗号アルゴリズム実装・外部の非標準暗号ライブラリの追加は禁止
- 鍵導出はPBKDF2、暗号化はAES-256-GCMで統一する

### 入力・出力のサニタイズ
- Tiptapエディタからのコンテンツ保存・表示時はHTMLサニタイズを行う（XSS対策）
- ユーザー入力をそのままDBクエリに渡さない（Prismaのパラメータ化を使用）

---

## コーディング規約

- TypeScript: `any`禁止、厳格な型定義を行う
- ファイル/フォルダ構成: Next.js App Routerの規約に従う（`app/`, `app/api/`）
- コンポーネントは機能単位で `components/` 以下に配置
- DBアクセスは必ずPrisma経由、生SQLは原則禁止
- 環境変数は `.env.example` に必ず追加し、実際の値は `.env` に記載（コミットしない）

---

## Symbol SDK利用に関する注意

- ネットワーク（テストネット/メインネット）は環境変数 `NEXT_PUBLIC_SYMBOL_NETWORK` で切り替え可能にする
- ノードURLも環境変数化する（`NEXT_PUBLIC_SYMBOL_NODE_URL`）
- 開発・テストは常にテストネットを使用する。メインネット対応はPhase 8以降、明示的指示があるまで行わない
- トランザクション署名・送金処理はすべてクライアントサイドコンポーネント内で実装する

---

## 通知機能


- ブラウザ通知はWeb Notifications API + Service Workerで実装する
- Firebase / Pusher / OneSignal 等の外部サービスは使用しない
- 通知のトリガーはサーバー側のポーリング結果（投げ銭着金検知、コメント投稿など）とする
- 通知許可はユーザーが明示的に許可した場合のみ発火する

## テスト

- 各フェーズ完了時に最低限の動作確認手順を `/docs/progress.md` に記載する
- 暗号化/復号、トランザクション作成などロジック部分は可能な範囲でユニットテストを作成する（Vitest推奨）

---

## 禁止事項まとめ

- 秘密鍵・ニーモニック・ウォレットパスフレーズのサーバー送信・保存・ログ出力
- 独自暗号アルゴリズムの実装
- 仕様書にないフェーズの先取り実装
- Phase 5/6実装後の無断commit/push
- メインネットへの接続（明示的指示があるまで）
